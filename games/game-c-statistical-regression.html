Game C • Statistical regression track

Building a valid statistical model with industry data on functional requirements and related development effort

Select the appropriate subset of ISBSG industry data, apply outlier removal, and build a valid linear regression model that relates functional size (COSMIC Function Points) to development effort.

Learning objective

Teach students how to construct and evaluate a linear regression model from industry benchmark data by selecting appropriate project subsets, handling outliers, and interpreting model fit statistics to judge whether an estimation model is valid.

How the game works

  1. Choose a data subset from the ISBSG database (90 projects filtered by application type, development type, or language).
  2. Review the descriptive statistics of the subset.
  3. Apply the Grubbs outlier test and remove flagged projects from the dataset.
  4. Fit a simple linear regression: Effort (hrs) = b0 + b1 × Size (COSMIC CFP).
  5. Evaluate model quality: examine R², MMRE, and the final sample size.
  6. Decide whether the model is valid for use in estimation and submit your assessment in SimSE.

What drives model quality

  • Sample size (N) - the number of projects used after outlier removal
  • R-square (R²) - proportion of effort variance explained by size
  • MMRE - Mean Magnitude of Relative Error; measures prediction dispersion
  • Subset homogeneity - filtering by type or language reduces variability
  • Outlier removal - Grubbs test applied to both size and effort distributions

Dataset and subsets

The game uses a database of 90 ISBSG projects measured in COSMIC Function Points. Each subset is defined by one or more filter criteria applied to the full dataset.

Model evaluation criteria

Three statistics are used to assess whether the regression model is suitable for software effort estimation.

Statistic Definition Interpretation
R² (R-square) Proportion of effort variance explained by functional size Higher is better. Values above 0.70 indicate a strong size-effort relationship.
MMRE Mean Magnitude of Relative Error across all projects in the model Lower is better. Values below 0.25 are generally considered acceptable.
N Number of projects used in the final regression after outlier removal A larger sample produces more reliable estimates. Very small N reduces model generalizability.

Outlier removal

Before fitting the regression, outliers are identified using the Grubbs test (significance level 0.05, two-sided) applied independently to the size and effort distributions. Projects flagged as outliers are excluded and the model is refitted on the remaining data.

Regression model form

All models in this game use a simple linear regression of the form:

Effort (hrs) = b0 + b1 × Size (CFP)

where Size is measured in COSMIC Function Points (CFP) and Effort in hours.

Teaching notes

This game develops students' ability to critically evaluate regression-based estimation models and understand how data subsetting affects model validity.

Suggested use in class

  • Introduce the concept of homogeneous data subsets before running the game
  • Discuss why filtering by application type, development type, or language can improve model fit
  • Compare R² and MMRE values across subsets and explain the differences
  • Highlight cases where outlier removal drastically reduces N and discuss the trade-off
  • Identify which subsets produce models suitable for estimation and justify the criteria used

What to evaluate

  • Ability to interpret R² and MMRE in the context of estimation quality
  • Understanding of the effect of subset selection on model homogeneity and fit
  • Correct application of outlier detection and its impact on the regression results
  • Judgment on whether a model with given N, R², and MMRE is valid for practical use
