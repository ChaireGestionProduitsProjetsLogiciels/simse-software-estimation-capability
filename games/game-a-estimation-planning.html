Game A • Estimation planning track

Function Points sizing of functional requirements and statistical skills issues for data analysis

Simulate the process of constructing a linear regression estimation model by selecting the right team members and the correct sequence of analytical steps for each of two historical datasets.

Resources

30 In-House Projects, then 100 ISBSG external projects.

Action

Select the right employee profile and the correct pair of statistical step sequences for each resource.

Outcome

Score based on employee skill fit and correctness of the chosen sequences.

Learning objective

Teach students how to plan a software estimation effort by selecting team members with the appropriate skills in software size measurement and linear regression statistics, and by ordering the correct analytical steps to build a valid size-effort model from historical project data.

How the game works

  1. Select one employee from six available profiles based on their measurement and statistics skills.
  2. Work on two resources, one after the other: 30 In-House Projects, then 100 ISBSG external projects.
  3. For each resource, choose the correct pair of statistical step sequences required to complete the regression model.
  4. End the game to see the final score across both rounds.

The same employee cannot be selected for both resources.

Employee selection criteria

Each of the six employee profiles is defined by two visible attributes:

  • Measurement Skill - whether the employee can perform software size measurement
  • Statistics Level - their competency in linear regression statistics: Top, Fair, or Low

Selecting the right profile for each resource type is essential for a correct estimation plan.

Statistical steps

Sequences are represented by combinations of letters such as BDAC. Each letter stands for an analytical step needed to build and interpret the regression model.

Code Step Description
A Outlier Identify projects too large that may mislead interpretation of the model's strength.
B R-square (R²) Measure the strength of the size-effort relationship.
C Measurement Apply knowledge of early software sizing and prepare the data.
D Descriptive Compute basic statistics describing the quantitative sample data.
E MMRE Analyse the dispersion of projects across the regression model using the Mean Magnitude of Relative Error.

For each resource, the player selects two sequences from the ten options below. The correct pair depends on the resource type and the skills of the selected employee.

Teaching notes

This game introduces students to the planning dimension of software estimation: who should be involved and what analytical process should be followed before any estimate is produced.

Suggested use in class

  • Discuss the differences between in-house project data and industry benchmark data (ISBSG)
  • Explain why measurement skill and statistics competency matter for building a valid model
  • Have students justify which employee profile fits each resource type and why
  • Walk through the logic of each step (A through E) before students select sequences
  • Compare the sequences chosen for each resource and discuss how the data source influences the required steps

What to evaluate

  • Ability to match employee skills to resource requirements
  • Understanding of the role of each analytical step in building a regression model
  • Correct selection and ordering of statistical steps per resource
  • Recognition that different data sources may require different analytical approaches
