Estimation Planning Questionnaire

Function Points sizing of functional requirements and statistical skills issues for data analysis - Game A

Game narrative
This is a learning game designed to simulate the process of constructing a linear regression estimation model in a software engineering context.

As a player, you have to:

  1. Select the right employee (from six available profiles) based on their skills in measurement and statistics.
  2. Work on two resources, one after the other: 30 In-House Projects, then 100 ISBSG Data.
  3. For each resource, choose the correct pair of task sequences required to complete the estimation model.

You can choose among different task sequences, each represented by a combination of letters such as BDAC. These letters stand for the analytical steps needed to build and interpret the regression model:

Code Step Description
A Outlier Projects too large that may mislead interpretation of the model's strength.
B R-square (R²) Strength of the size-effort relationship.
C Measurement Knowledge of early sizing and data preparation.
D Descriptive Basic statistics describing the quantitative sample data.
E MMRE Analyse the dispersion of the projects across the regression model.
